Tahoe Open Data: Data Policy and Developer Guide

How to use and access data published on tahoeopendata.org.

Draft   v0.5 · Jul. 7, 2026 · TRPA GIS (gis@trpa.org) · Disclaimer per tahoeopendata.org/pages/policy

This page covers the terms that apply to TRPA open data and how to pull it programmatically. By using data from tahoeopendata.org you accept the policy below.

Data and Use Policy

How data is delivered

Content is served two ways. Knowing which applies tells you whether it can be queried or only downloaded.

DeliveryContentAccess
ArcGIS services (maps.trpa.org) Vector map and feature services, including the nine DataDownloader themes. ArcGIS REST API, or the download options on each item page.
Box file downloads Large rasters and files (DEMs, bathymetry, Snow PALM, vegetation ecobject). Direct file download from the item's link. Not available through the API.

Program and indicator data (EIP, Parcel Tracker, Development Rights, thresholds) lives in a separate platform, LT Info, described in the Developer Guide.

Use and attribution

There is no restriction on the use or distribution of the data contained in this portal. However, please acknowledge the Tahoe Regional Planning Agency in any products or applications that reference this data. A suggested credit:

Source: Tahoe Regional Planning Agency (TRPA), Tahoe Open Data,
tahoeopendata.org. Accessed [date].

Some datasets include partner data (county assessors for parcels; USFS and NRCS for soils and vegetation). Keep any added credits noted in the item metadata.

Fair use

These are shared public services. Please:

  • Keep request rates reasonable; avoid aggressive parallel scraping.
  • For bulk needs, download the full dataset once instead of paging the API in a loop.
  • Filter server-side and request only the fields and extent you need.
  • Cache results locally.
  • Sync incrementally. Track a layer's edit date and only re-pull records that changed instead of re-downloading the whole layer. Where a layer exposes an edit-date field or editingInfo.lastEditDate, filter on it with a where clause; for LT Info, use the nightly-updated endpoints on a daily schedule.

TRPA does not guarantee uptime or a request quota for public services and may throttle traffic that affects others. For high-volume or operational needs, contact TRPA GIS.

Sensitive data

  • The public parcel download includes parcel identifiers, addresses, and geometry. It does not include owner names or assessed values. Nevada county parcels are not offered; contact the county assessor.
  • Some wildlife datasets show locations of protected or sensitive species. Use them responsibly and do not use them to disturb wildlife. TRPA may generalize or restrict these layers.

Updates

Datasets refresh on different schedules. Where a cadence is set, it appears on the item page. Applications that embed TRPA data should re-fetch periodically rather than rely on a stored copy.

Developer Guide

Access one dataset

Each map or feature service item offers two endpoints from its page: a GeoService (the ArcGIS REST layer, with fields and query tools) and a GeoJSON link (features ready for a web map). For a packaged file, use the download options on the item page.

GeoService  https://maps.trpa.org/server/rest/services/DataDownloader_Transportation/MapServer/14
GeoJSON     .../MapServer/14/query?where=1=1&outFields=*&f=geojson

ArcGIS REST API

No key is required for public services. Base URL:

https://maps.trpa.org/server/rest/services

The download themes are the nine DataDownloader services (Contours, Impervious, Parcels, PlanningandJurisdictions, Recreation, Shorezone, SoilsandHydro, Transportation, Wildlife). Responses come as json, geojson, or pbf.

# list the layers in a service
https://maps.trpa.org/server/rest/services/Datadownloader_PlanningandJurisdictions/MapServer?f=json

# query a layer as GeoJSON
.../DataDownloader_Transportation/MapServer/14/query?where=1=1&outFields=*&f=geojson

# filter by attribute, return only needed fields
.../DataDownloader_Parcels/MapServer/0/query?where=JURISDICTION='El Dorado'&outFields=APN,APO_ADDRESS&f=json

Services return up to 2,000 records per request. Page through larger layers with resultOffset and resultRecordCount:

.../MapServer/0/query?where=1=1&outFields=*&resultOffset=0&resultRecordCount=2000&f=geojson
.../MapServer/0/query?where=1=1&outFields=*&resultOffset=2000&resultRecordCount=2000&f=geojson

LT Info Data Center

Lake Tahoe Info is TRPA's program-data platform and the system of record for EIP projects and performance measures, Parcel Tracker attributes, Development Rights, threshold evaluations, and more. It is built on the open-source ProjectFirma platform (hosted by ESA). It offers a token-based tabular API and a GeoServer for spatial layers.

Tabular API (token required)

Register a Keystone account, generate a token on the Web Services page, and pass it in each request. Treat the token like a password; do not embed it in public or client-side code. The legacy SOAP interface is no longer maintained.

https://www.laketahoeinfo.org/WebServices/{ServiceName}/{FORMAT}/{token}[/{param}]
# FORMAT = CSV or JSON.  {param} = EIP Project #, OrganizationID, or Indicator ID where noted.

https://www.laketahoeinfo.org/WebServices/GetProjects/JSON/{token}
https://www.laketahoeinfo.org/WebServices/GetTransactedAndBankedDevelopmentRights/JSON/{token}
ThemeEndpoints
EIP and projects GetProjects, GetProject, GetProjectExpenditures, GetReportedProjectAccomplishments, GetEIPIndicators, GetEIPHomepageStats
Parcels GetAllParcels, GetParcelsByLandCapability, GetParcelIPESScores, GetDeedRestrictedParcels
Development Rights GetBankedDevelopmentRights, GetTransactedAndBankedDevelopmentRights, GetDevelopmentRightTransactions, GetDevelopmentRightPoolBalanceReport
Other GetOrganizations, GetThresholdEvaluations, GetTransitMonitoringData, GetAssignedMooringTags

GeoServer (WFS / WMS)

LT Info spatial layers are served by GeoServer and can be added in ArcGIS Pro or QGIS over WFS or WMS, then exported to Shapefile or File GDB.

https://mapserver.laketahoeinfo.org/geoserver

Layers include AllParcels, TrackedParcels, AllStormwaterCatchments, ParcelDevelopmentRightTransfers, Project, vEIPProjectLocationDetail, and mooring layers.

Code snippets

Leaflet

const map = L.map('map').setView([39.09, -120.03], 10);
L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png').addTo(map);

const url = 'https://maps.trpa.org/server/rest/services/'
  + 'DataDownloader_Transportation/MapServer/14/query'
  + '?where=1=1&outFields=*&f=geojson';

fetch(url).then(r => r.json()).then(geojson => L.geoJSON(geojson).addTo(map));

Python (GeoPandas), ArcGIS layer

import geopandas as gpd

url = ("https://maps.trpa.org/server/rest/services/"
       "DataDownloader_SoilsandHydro/MapServer/5/query"
       "?where=1=1&outFields=*&f=geojson")

watersheds = gpd.read_file(url)
print(watersheds.shape)

Python (requests), LT Info tabular API

import requests

token = "YOUR_TOKEN"   # keep this private
url = f"https://www.laketahoeinfo.org/WebServices/GetProjects/JSON/{token}"

projects = requests.get(url, timeout=120).json()
print(len(projects))
